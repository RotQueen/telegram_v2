# Telegram Project Relay Bot

Бот-посредник для связывания чатов заказчиков и исполнителей по проектам. Работает в нескольких групповых чатах и пересылает сообщения между парами чатов для каждого проекта.

## Функции
- Создание проекта с привязкой чата исполнителей (`/create_project <slug>`).
- Привязка чата заказчиков (`/bind_customer <slug>`).
- Получение информации о привязке текущего чата (`/project_info`).
- Список проектов для админа (`/list_projects`).
- Отвязка чата или деактивация проекта (`/unlink_project <slug>`).
- Пересылка текста, документов, фото, аудио, видео и голосовых сообщений между чатами.

## Запуск на Railway
1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Задайте переменные окружения:
   - `BOT_TOKEN` — токен Telegram-бота (обязателен).
   - `ADMIN_USER_ID` — Telegram ID администратора (опционально, по умолчанию 5386753143 — @askeditme).
   - `DATABASE_PATH` — путь к SQLite-файлу (опционально, по умолчанию `./projects.db`).
3. Запустите бота (используется long polling, вебхуки не нужны):
   ```bash
   python main.py
   ```
4. Размещая на Railway, укажите команду запуска `python main.py` и задайте переменные окружения в настройках сервиса.

## Структура
- `config.py` — загрузка конфигурации из переменных окружения.
- `storage.py` — слой хранения проектов на SQLite (можно заменить другим хранилищем).
- `main.py` — инициализация бота, регистрация хэндлеров и запуск long polling.
- `requirements.txt` — зависимости (python-telegram-bot 20.x).
